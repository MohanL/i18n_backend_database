h1. A Database Backend For Rails I18N

Store your translations in the database, rather than yaml files.  As you tag items with i18n.t() throughout your code base, all untranslated items are marked and added to the database.  An admin panel is provided so translators can quickly translate untranslated text.  All keys are cached, and an alternate cache store can be provided.


h2. Installation

<pre>
  <code>
    script/generate i18n_backend_database # add migration
    rake db:migrate                       # migrate
    rake i18n:populate:all                # populate locales (list from Java's LocaleServiceProvider)
  </code>
</pre>


In config/initialisers/i18n.rb

<pre>
  <code>
    I18n.backend = I18n::Backend::Database.new # registers the backend
    I18n.backend.cache_store = :memory_store   # optional: specify an alternate cache store
  </code>
</pre>

In config/routes.rb to register admin panel routes

<pre>
  <code>
    map.from_plugin 'i18n_backend_database'
  </code>
</pre>
